# AICall — AI+在线会诊系统 项目说明书

## 一、项目概述

### 1.1 项目名称

**AICall** — AI 赋能的在线多学科会诊系统

### 1.2 项目定位

面向医院场景，为患者提供在线发起多学科会诊（MDT）的服务平台。系统分三个端：

- **用户端**：患者/家属使用
- **医生端**：会诊专家使用
- **管理端**：医院运营使用

全流程深度融入 AI 能力，提升会诊效率与质量。

### 1.3 核心价值

| 端     | 核心价值                                                     |
| :----- | :----------------------------------------------------------- |
| 用户端 | 降低会诊发起门槛，AI 辅助预问诊、资料整理，会诊后提供通俗化解读 |
| 医生端 | AI 预分析病历、病灶勾勒、辅助生成报告，大幅缩短准备和书写时间 |
| 管理端 | AI 趋势预测、自动质控、满意度分析，数据驱动运营决策          |

### 1.4 支付说明

采用**模拟支付**模式：

- 系统正常计算费用并展示明细
- 用户确认支付时模拟支付成功，无需真实扣款
- 后续流程正常流转

### 1.5 用户端身份说明

用户端**无需登录和实名认证**，直接以访客身份发起会诊。

- 通过会诊申请时填写的姓名和手机号标识身份
- 会诊完成后通过手机号查询会诊记录

## 四、AI 调用架构

### 4.1 全部使用云端 API

所有 AI 能力均通过硅基流动 API 调用，无需本地部署 AI 服务。

| AI 能力           | 调用方式               | 说明                  |
| :---------------- | :--------------------- | :-------------------- |
| LLM 对话/文本生成 | 硅基流动 Chat API      | DeepSeek、Qwen 等模型 |
| Embedding 向量化  | 硅基流动 Embedding API | BGE-M3 等模型         |
| OCR 识别          | 硅基流动 OCR API       | 病历、化验单文字提取  |
| 语音识别 (ASR)    | 硅基流动 ASR API       | 会诊实时语音转文字    |
| 影像 AI 分析      | 硅基流动影像 API       | 病灶检测、量化分析    |

### 4.2 调用链路

```
业务代码 → module/ai/service/
│
├── 文本生成/对话 ──── LangChain4j ──── 硅基流动 Chat API
│
├── Embedding ──────── LangChain4j ──── 硅基流动 Embedding API
│ │
│ 存入 Qdrant
│
├── RAG 检索增强 ──── Qdrant 检索 → 拼接上下文 → Chat API
│
├── OCR 识别 ──────── 硅基流动 OCR API
│
├── 语音识别 ──────── 硅基流动 ASR API
│
└── 影像分析 ──────── 硅基流动影像 API
```



### 4.3 Qdrant 部署选项

| 方案         | 说明                           |
| :----------- | :----------------------------- |
| Qdrant Cloud | 官方托管，免运维，按量付费     |
| 自建 Docker  | 本地或服务器部署，数据全量本地 |

> 建议开发阶段用 Docker 自建，生产环境根据数据合规要求选择。

### 4.4 数据安全策略

| 措施           | 说明                             |
| :------------- | :------------------------------- |
| 调用前统一脱敏 | 去除姓名、身份证号、手机号、住址 |
| 保留医学信息   | 年龄、性别、症状、检查指标等     |
| AI 输出标注    | 必须标注"由AI生成，仅供参考"     |
| 最终决策权     | 归属医生，AI 不可替代            |

## 五、用户端功能清单

> **身份说明**：用户端无需登录注册，以访客身份直接使用。发起会诊时填写姓名和手机号作为身份标识和联系方式，会诊完成后通过手机号查询会诊记录和报告。

### 5.1 发起会诊

**填写基本信息**

- 输入患者姓名、手机号
- 选择与患者关系（本人 / 家属）
- 如是家属，填写本人姓名

**AI 预问诊**

- 用自然语言描述病情（如"左侧乳房胀痛"）
- AI 对话式追问起病时间、症状性质、加重因素、既往病史等
- 问诊结束后，AI 自动生成结构化《病情摘要》草稿（主诉、现病史、既往史、核心问题），用户确认或修改

**资料上传**

- 支持拍照、相册、本地文件上传影像、化验单、病理报告（含 DICOM 格式）
- 上传后 AI 实时审核资料质量，提示模糊、反光、缺项等问题

**选择会诊**

- 系统根据预问诊结果智能推荐会诊类型（单学科 / 多学科 MDT）
- 用户也可自选科室或指定医生
- 阅读并勾选《知情同意书》

**费用与支付**

- 清晰展示费用构成明细及合计金额
- 点击"确认支付"，系统模拟支付成功，无需真实扣款
- 跳转"支付成功"页，显示申请已提交

### 5.2 会诊进度跟踪

- 通过手机号查询会诊状态
- 状态节点：

申请已提交 → 资料审核中 → 专家确认中 → 会诊已排期 → 待会诊 → 会诊进行中 → 会诊已完成

- 每个节点变更时短信推送提醒
- 专家确认前可取消申请，模拟退款到账

### 5.3 参与会诊

- 预约时间前 15 分钟短信提醒，通过链接进入候诊室
- 进入会诊室前引导检测摄像头、麦克风、网络
- 高清视频会诊
- AI 实时字幕：医生语音实时转文字，医学术语旁自动弹出通俗解释
- 医生共享屏幕讲解影像时，用户端同步观看
- AI 后台实时提炼关键对话，会诊结束生成带时间戳的要点纪要，点击可回看对应视频片段

### 5.4 会诊后

- **双版报告**：医生签发专业版《会诊意见书》PDF，AI 同步生成通俗《患者版健康小结》
- 会诊视频回放，支持点击纪要节点跳转对应片段
- **AI 就医助手**：针对会诊建议（如"做核磁共振"），一键咨询后续流程
- **AI 智能随访**：第 3、7、30 天自动短信推送康复问卷链接，AI 判断恢复异常时提醒复诊
- 根据诊断结果自动推送个性化健康宣教内容
- 对专家和整体服务进行星级评分与文字评价

## 六、医生端功能清单

### 6.1 登录与工作台

- 账号密码登录（由管理员后台创建账号）
- 统一任务池：待处理 / 已处理会诊列表，按紧急度、疾病类型标签分类，急诊红标
- **AI 预分析速览**：点进病例优先展示：
  - 一句话病情摘要
  - 核心异常指标（影像关键帧截图 + 化验异常值红标）
  - AI 初步诊断推测及依据
  - 资料齐备度评分（缺项提醒）

### 6.2 会诊处理

- 在线详阅完整病历、原始影像（内置 DICOM 浏览器，支持窗宽窗位调节、多序列联动、测量标注）
- 资料齐全则确认参与，选择可排期时间
- 资料不全则填写退回原因（可选用 AI 生成的退回事由建议）
- 系统自动检测日程冲突，锁定排期时间

### 6.3 多学科会诊室

- 多方高清音视频通话
- 主持人可控制发言人权限
- 屏幕共享（影像、病历、PPT）
- 多人互动批注（激光笔、圈画标记）
- **AI 增强阅片**：打开 CT/MRI 影像，AI 自动勾勒病灶轮廓，量化病灶大小、体积、密度；多次检查影像同屏对比变化
- **临床决策辅助**：侧边栏查询最新临床指南；检索脱敏病历库中的相似病例及其治疗方案与预后

### 6.4 AI 报告生成与管理

- 会诊结束后，AI 自动生成完整《会诊意见书》草稿：
  - 会诊经过（时间、参会专家、讨论过程）
  - 诊断分析讨论
  - 最终诊断（含 ICD 编码）
  - 分条治疗建议（用药、手术、随访计划等）
- 医生在线修改审核，手写电子签名
- 一键生成带医院水印的正式 PDF，签发后自动同步至用户端
- 内置各科室报告模板，支持自定义模板

### 6.5 我的患者

- 对参与会诊的患者进行分组管理
- 查看患者后续康复问卷填写情况
- 查看同一患者历次会诊记录

## 七、管理端功能清单

### 7.1 业务监控驾驶舱

- 实时数据大屏：
  - 今日会诊总数、进行中 / 待接诊 / 已完成量
  - 平均响应时间、专家上线率
  - 患者区域分布地图
- **AI 趋势预测**：预测下周会诊量高峰时段与热门科室
- **超时预警**：待审核、待接诊环节超时自动标红告警

### 7.2 分诊与调度

- 分诊规则配置：按疾病类型、科室设置自动分配规则
- 专家排班管理：设置各专家组 MDT 排班时段
- 诊室资源管理：设置线上会诊室数量与开放时段
- 负荷监控：各组专家任务量不均时自动预警

### 7.3 AI 质量管理

- **AI 报告质控**：
  - 每份签发报告自动检查完整性（签名、日期、随访建议）
  - 检查规范性（术语统一、格式合规）
  - 检查一致性（结论与建议逻辑是否矛盾）
  - 生成每日质控报告，低分报告退回修改
- **绩效统计**：每位专家会诊量、平均时长、按时完成率、退回次数、好评率
- **满意度分析**：AI 分析患者评价文本，提炼高频关键词与情感倾向

### 7.4 费用与财务

- 费用模板配置：不同会诊类型定价、费用构成项设置
- 交易记录管理：查看所有模拟支付订单，支持模拟退款操作
- 财务统计报表：按日 / 周 / 月统计收入

### 7.5 系统管理

- 角色权限管理：超级管理员、运营、质控、财务等角色，细粒度权限控制
- 医生账号管理：创建、启用 / 禁用
- 三端用户账户查询
- 操作日志审计（不可删除）
- 敏感数据脱敏规则配置
- 消息模板、知情同意书模板、评价表单等自定义配置

## 九、核心业务流程

### 9.1 会诊主流程

```
用户填写姓名手机号
↓
AI 预问诊（对话式采集病情）
↓
上传资料（影像、化验单、病理报告）
↓
AI 审核资料（模糊、缺项提示）
↓
确认会诊类型（单学科/MDT，自选或推荐）
↓
费用计算 → 模拟支付
↓
管理员/系统分诊
↓
医生审核资料
├── 齐全 → 医生确认排期 → 短信通知用户
└── 不全 → 退回说明原因 → 短信通知补充 → 重新提交
↓
到点进入会诊室
↓
音视频会诊（AI 实时字幕 + 要点速记）
↓
AI 生成报告草稿
↓
医生审核修改 → 电子签名 → 生成正式 PDF
↓
报告同步用户（双版：专业版 + 患者版）
↓
AI 智能随访（第 3/7/30 天）
```

### 9.2 模拟支付流程

```
用户确认会诊
↓
系统计算费用（专家费 + 平台服务费）
↓
展示费用明细
↓
用户点击"确认支付"
↓
调用后端模拟支付接口
↓
生成支付成功记录（交易号、时间戳）
↓
返回支付成功
↓
会诊状态变更为"已支付，待审核"
```

### 9.3 用户端身份标识流程

```
用户打开系统 → 直接使用（无需登录）
↓
发起会诊时 → 填写姓名 + 手机号
↓
提交后生成会诊编号
↓
查询会诊 → 输入手机号 → 查看关联会诊列表
↓
参与会诊 → 短信中的链接直接进入会诊室
↓
查看报告 → 短信链接或手机号查询

t---
```

### 12.1 性能指标

### 12.2 安全与合规

| 要求         | 说明                             |
| :----------- | :------------------------------- |
| 全站 HTTPS   | 数据传输加密                     |

| AI 数据脱敏  | 调用 API 前去除身份信息          |

| 最终决策权   | 归属医生，AI 不可替代            |
| 防刷限制     | 用户端接口需验证码、频次限制     |
