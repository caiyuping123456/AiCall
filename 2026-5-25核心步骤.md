# AICall 三端功能核心步骤

## 一、患者端（User H5）—— `/user`

### 1.1 注册/登录
1. 打开 H5 页面 → 首页 → 点击"注册"
2. 填写姓名、手机号、性别、年龄、身份证号、密码 → 提交注册
3. 注册成功后跳转登录页 → 输入手机号 + 密码 → 登录
4. 登录成功拿到 Token，进入首页

### 1.2 发起会诊（核心流程）

#### 步骤 1：选择预问诊方式
1. 首页点击"发起会诊" → 进入预问诊方式选择页
2. 系统自动创建会诊草稿（status=0）
3. 选择两种方式之一：
   - **对话模式**：与 AI 分诊护士进行多轮对话，逐步描述病情
   - **表单模式**：手动填写结构化表单（主诉、起病时间、症状描述、既往史、过敏史）

#### 步骤 2：AI 预问诊（对话模式）
1. AI 护士发起首轮问候："您好，我是 AICall 分诊护士，请问您哪里不舒服？"
2. 患者逐轮回复，AI 持续追问关键细节（部位、性质、时长、诱因等）
3. AI 判断信息足够后提示"问诊完成" → 自动生成病情摘要

#### 步骤 2'：填写病情（表单模式）
1. 填写主诉（必填）、起病时间、症状描述、既往史、过敏史
2. 点击"提交" → 后端调用 AI 生成摘要

#### 步骤 3：查看病情摘要
1. 展示 AI 生成的五维摘要：主诉、现病史、既往史、过敏史、初步印象
2. 患者确认后进入下一步"上传资料"

#### 步骤 4：上传病历资料
1. 点击"上传"选择图片/PDF 文件
2. 上传至 MinIO，后端调用视觉模型 OCR 识别文字内容
3. OCR 结果向量化存入 Qdrant 向量数据库

#### 步骤 5：选择会诊类型
1. **单学科会诊**：¥500.00，一位专家看诊
2. **多学科 MDT 会诊**：¥1,500.00，多位专家联合看诊
3. 选择后后端计算费用并更新 consultation 的 type 和 fee 字段

#### 步骤 6：确认支付
1. 展示会诊编号、类型、费用
2. 点击"确认支付" → 生成支付单 → 更新 payment_status=1 → status=2（专家确认中）
3. 跳转"支付成功"页面

### 1.3 查询会诊
1. 首页点击"查询会诊" → 列表显示该患者所有会诊记录
2. 点击某条记录可查看详情/摘要/状态

### 1.4 随访
1. 首页/底部导航 → "我的随访"
2. 列表显示所有随访计划（3天、7天、30天等）
3. 点击某期随访 → 填写 AI 生成的问卷（单选、多选、文本）
4. 提交后 AI 分析回答，异常情况自动通知医生

### 1.5 评价
1. 会诊完成后进入"会诊评价"页面
2. 对医生服务进行评分

### 1.6 通知中心
- 查看系统推送的通知消息（会诊状态变更、随访提醒等）

## 二、医生端（Doctor Desktop）—— `/doctor`

### 2.1 登录
1. 医生账号由管理员在后台预先创建（用户名格式：`doctor_` + 随机 6 位字符）
2. 输入用户名 + 密码 → 登录 → 进入工作台

### 2.2 工作台（Dashboard）
1. 顶部三个统计卡片：
   - **待审核会诊**：状态为"专家确认中"（status=2）的会诊数
   - **报告编辑中**：正在编辑/待质控的报告数
   - **待质控**：需要质控的报告数
2. 下方展示"最近会诊"列表，点击可进入详情

### 2.3 接诊确认（核心）

#### 步骤 1：查看待审核会诊
1. 工作台点击"待审核会诊" → 跳转会诊列表（status=2 筛选）
2. 或直接进入"会诊列表"页面，可按状态筛选、搜索

#### 步骤 2：查看会诊详情
1. 点击某条会诊 → 进入详情页
2. 查看四个 Tab：
   - **患者信息**：姓名、年龄、性别、科室、主诉
   - **病情摘要**：AI 生成的五维摘要
   - **上传资料**：患者上传的文件列表及 OCR 识别结果
   - **问诊记录**：AI 预问诊的对话历史

#### 步骤 3：确认或拒绝
1. 当 status=2 时，页面显示两个按钮：
   - **确认接诊**：点击即确认，consultation 状态变为 3（已排期）
   - **拒绝**：输入拒绝原因后点击确认

#### 步骤 4：生成 AI 报告
1. status=3（已排期）时，点击"生成 AI 报告"
2. 后端根据患者信息、AI 摘要、对话记录、OCR 结果调用 LLM 生成结构化报告
3. 报告包含：主诉、现病史、既往史、检查所见、诊断意见、分析说明、建议、随访建议

### 2.4 编辑/质控/签发报告

#### 步骤 1：编辑报告
1. 详情页 status=4 时，点击"编辑报告"进入报告编辑器
2. 可逐字段编辑：主诉、现病史、既往史、检查所见、诊断意见、分析说明、建议、随访建议
3. 点击"保存"保存草稿，点击"提交质控"进入质控流程

#### 步骤 2：AI 质控
1. 后端从完整性、规范性、一致性三个维度打分（0-100）
2. 展示各项得分和具体问题列表
3. 分数低于阈值需修改后重新提交

#### 步骤 3：签名签发
1. 质控通过后（status=1 且 qcResult.status=1），显示"签名签发"按钮
2. 点击签发 → 报告状态变为"已签发"，consultation 进入"已完成"

### 2.5 音视频会诊（会诊室）
1. 详情页 status=3 或 4 时，点击"进入会诊室"
2. 初始化 TRTC 房间，建立音视频连接
3. 功能：
   - **摄像头/麦克风**：开关控制
   - **屏幕共享**：分享桌面/文档
   - **实时字幕**：浏览器 Web Speech API 语音识别，通过 WebSocket 广播给所有参会者
4. 会诊结束后点击"结束会诊" → 自动生成 AI 会议纪要 + 创建随访计划

### 2.6 随访查看
- 会诊详情的"随访记录"Tab 可查看患者的随访回答和 AI 分析

### 2.7 通知中心
- 查看系统通知

## 三、管理员端（Admin Panel）—— `/admin`

### 3.1 登录
1. 访问管理后台 → 登录页
2. 输入账号 `admin` + 密码 `admin123`（BCrypt 加密存储）
3. 登录成功进入仪表盘

### 3.2 数据仪表盘（Dashboard）
1. 顶部统计卡片：会诊总数、本月新增、本周新增、已收金额
2. 图表区域（ECharts）：
   - **科室分布**：饼图，各科室会诊数量占比
   - **状态分布**：饼图，各状态会诊数量占比
   - **近 30 天趋势**：折线图，每日会诊增长趋势
3. **医生工作量排名**：表格，按会诊数排名

### 3.3 医生管理

#### 列表
1. 搜索：按姓名/科室/手机号模糊搜索
2. 列表：分页展示，含姓名、职称、科室、手机号、状态开关
3. **启用/禁用**：直接切换开关控制医生 account 状态

#### 新增医生
1. 点击"新增医生" → 弹窗填写姓名、职称、科室、手机号、密码
2. 提交后系统自动生成用户名（`doctor_` + 随机串），密码 BCrypt 加密

#### 编辑医生
1. 列表点击"编辑" → 弹窗修改姓名、职称、科室、手机号、简介

#### 医生详情
1. 显示基本信息：账号、姓名、职称、科室、手机号、简介、会诊数
2. **排班管理**：
   - 可按日期筛选排班记录
   - **新增排班**：选择日期 + 开始时间 + 结束时间
   - **删除排班**：点击删除按钮

### 3.4 会诊管理

#### 会诊列表
1. 按状态筛选：已提交/资料审核中/专家确认中/已排期/待会诊/会诊中/已完成/已取消/已退回
2. 按患者姓名/会诊编号搜索
3. 分页列表展示：会诊编号、患者、科室、状态、费用、支付状态、创建时间

#### 会诊详情
1. 展示基本信息（会诊编号、患者、年龄、性别、科室、状态、费用、支付状态）
2. 展示主诉和 AI 摘要
3. **指派医生**（status < 3 时可用）：
   - 点击"指派医生"按钮 → 弹窗
   - 搜索并选择医生（支持多名），每位医生指定角色（主持人/普通专家）
   - 提交后 consultation_doctor 表写入记录，会诊状态推进至 2（专家确认中）
4. 查看已指派医生的确认状态（待确认/已确认/已拒绝）
5. 查看上传资料列表
6. 查看会诊纪要和录像（完成后）

#### 取消会诊
1. status ≠ 6 且 status ≠ 7 时，显示"取消会诊"按钮
2. 输入取消原因 → 确认 → status 变为 7（已取消）

## 附录：会诊状态流转总览

```
患者端                          管理员端                     医生端
───────                        ────────                    ────────
0  草稿（创建会诊）
│
├─→ 对话/表单预问诊
│
├─→ 查看摘要 → 上传资料
│
├─→ 选择类型 → 支付
│       │
└───→ 2  专家确认中  ←─── 管理员指派医生 ───→ 医生查看待审核列表
                      (POST /consultations/{id}/doctors)      │
                                                   ├─ 确认接诊 → 3 已排期
                                                   └─ 拒绝
                                                        │
                              3  已排期 ←─────────────────┘
                                                        │
                                                  生成 AI 报告
                                                        │
                              4  待会诊 ←─────────────────┘
                                                        │
                                                  进入会诊室
                                                        │
                              5  会诊中（TRTC 音视频）
                                                        │
                                                  结束会诊
                                                        │
                              6  已完成 ←─── 签发报告 ───┘
                                                        │
                              7  已取消 ←─── 管理员/医生取消
                              8  已退回 ←─── 管理员退回
```
