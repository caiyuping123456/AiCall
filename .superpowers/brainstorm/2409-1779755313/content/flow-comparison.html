用户挂号流程：旧 vs 新

对比前后端数据交互的变化

旧流程：多次后端写入 + 多次通知
Start前端
⬇ API: 创建草稿
Chat / Form🔔 通知医生
⬇ API: 更新摘要
Upload🔔 通知医生
⬇ API: 上传文件
Select Type🔔 通知医生
⬇ API: 支付
Success
❌ 医生收到多条重复通知
新流程：单次后端写入 + 单次通知
Start → Pinia state
Chat / Form / Summary → Pinia state
Upload → 临时存储 (返回 fileId)
Select Type → Pinia state
Pay → Pinia state
⬇ POST /consultations（一次调用）
Success🔔 仅1条通知
✅ 医生只收到1条通知
